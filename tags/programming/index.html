Thoughts on Rust Web-Development

2017-10-28
#rust  #webdev  #programming 
This entry will be a little scattershot, just some miscellaneous thoughts and impressions. Interacting with APIs It’s a little inconvenient to be making a lot of little structs for each feature of an API; where in JavaScript or Python you would have a dictionary, if you want type safety you need to make a struct with the proper fields. Strictly speaking you don’t need to; operating on untyped JSON is actually fairly straightforward, but when you know that the rest of your codebase isn’t going to crash the parts that aren’t guarenteed by the compiler niggle in your brain.
